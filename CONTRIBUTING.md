# Contributing to ForgeFlow

Thanks for investing time into ForgeFlow.

## Development Flow
1. Fork the repo and create a feature branch.
2. Keep changes modular and strictly typed.
3. Run `npm run lint`, `npm run typecheck`, and `npm test` before PR.

## Guidelines
- No telemetry.
- No new dependencies without justification.
- Keep files focused; avoid god files.
- Write tests for core logic, especially run commands and scanners.

## Pull Requests
Include a short summary, rationale, and test evidence.
